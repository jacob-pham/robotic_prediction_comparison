## Project Overview

This repo benchmarks trajectory prediction models on the ETH/UCY pedestrian dataset using the Social-GAN leave-one-out evaluation protocol. Each model (one per branch) is trained and evaluated on the same processed data splits. The current branch implements an SSM-based predictor using S4D.

I want to implement a potential-field-based navigation controller that uses the trajectory prediction model outputs for simple obstacle avoidance.

The goal is to add a controller/simulation module that:
1. Takes an ego agent's current position and goal position.
2. Uses the trained trajectory predictor to predict future pedestrian/obstacle positions.
3. Computes a navigation action using:
   - an attractive force toward the goal
   - repulsive forces away from predicted pedestrian positions
4. Rolls out the ego trajectory over time while avoiding predicted obstacles.
5. Produces simple plots showing:
   - ego start and goal
   - pedestrian observed/predicted trajectories
   - ego path generated by the potential field controller

## Commands

All commands are run from the project root.

**Preprocess data** (must be done before training any model):
```bash
python preprocess.py
```
Outputs `datasets_processed/eth/{train,val,test}.pt`. To switch scenes, change `scene_name` in `preprocess.py` (options: `eth`, `hotel`, `univ`, `zara1`, `zara2`).

**Train the SSM model:**
```bash
cd ssm && python train.py
```
Saves the best checkpoint to `ssm/checkpoints/best_model.pt` and a loss curve to `ssm/loss_curve.png`.

**Evaluate the SSM model:**
```bash
cd ssm && python test.py
```
Prints ADE and FDE on the test set, saves a prediction plot to `predictions.png`.

**Quick model shape check:**
```bash
cd ssm && python model.py
```

## Architecture

**Data flow:**
1. Raw `.txt` files in `datasets/{scene}/` → `preprocess.py` → `datasets_processed/{scene}/{split}.pt`
2. `.pt` tensors are `(N, 20, 2)` float32: N trajectory windows, 20 timesteps, (x, y) coords
3. Each trajectory is agent-centric: position at timestep 7 (last observed frame) is normalized to origin

**Sequence convention:** Every trajectory has 20 frames total — indices 0–7 are observed, 8–19 are the prediction target. During training and inference, future frames are zeroed out before being fed to the model. Loss is computed only on the future 12 frames.

**SSM model (`ssm/`):**
- `s4d.py` — standalone S4D implementation (no external S4 repo dependency); computes SSM convolution via FFT
- `model.py` — `TrajectoryPredictor`: linear input projection → 2× S4D blocks with pre-norm residuals → linear output projection
- `train.py` — training loop with Adam, MSE loss on future steps, val-loss-based checkpointing
- `test.py` — ADE/FDE evaluation and trajectory visualization

**Dataset structure:** Five scenes (eth, hotel, univ, zara1, zara2). Each scene folder acts as one leave-one-out fold: `test/` holds that scene's data, `train/` and `val/` hold the other four scenes split chronologically.
